# Діаграма послідовності

## Основний сценарій - реєстрація учасника

User
    ↓
Console UI
    ↓
EventService
    ↓
IEventRepository
    ↓
Перевірка правил домену
    ↓
IRegistrationRepository
    ↓
Успішна реєстрація
    ↓
Console Output


## Опис сценарію

1. Користувач обирає реєстрацію на подію.
2. Консольний інтерфейс отримує дані користувача.
3. EventService перевіряє бізнес-правила.
4. Репозиторій перевіряє існування події та кількість місць.
5. Створюється реєстрація.
6. Результат повертається в консоль.
