# Журнал завдань (Project Backlog) інформаційної системи "EventManager"

## 1. Ітерація №1 (Лабораторна робота №34)

### 1.1. Стратегічні цілі та інженерні завдання

* Проєктування та декомпозиція системи на принципах багатошарової архітектури (Clean Architecture).
* Формування ядра системи за методологією DDD (Domain-Driven Design) та інкапсуляція доменних інваріантів.
* Абстрагування шару доступу до даних за допомогою патерну Repository (In-Memory реалізація).
* Вертикальне наскрізне проєктування (Vertical Slice) базового функціонала системи.
* Розгортання інфраструктури безперервної інтеграції (CI) за допомогою інструментарію GitHub Actions.

### 1.2. Специфікація функціональних вимог

| Ідентифікатор | Функціональний компонент | Опис бізнес-логіки |
| --- | --- | --- |
| **FR-1.1** | Створення події | Ініціалізація сутності `Event` з обов'язковою валідацією початкових параметрів (назва, дата, ліміт місткості). |
| **FR-1.2** | Реєстрація учасника | Створення сутності `Registration` та зв'язування об'єктів `Participant` та `Event`. |
| **FR-1.3** | Валідація ресурсних обмежень | Автоматичний контроль доступної кількості місць на локації (`Venue.MaxCapacity`) перед фіксацією реєстрації. |
| **FR-1.4** | Моніторинг подій | Формування та виведення базового переліку наявних у системі подій через консольний інтерфейс. |

## 2. Ітерація №2 (Лабораторна робота №35)

### 2.1. Стратегічні цілі та інженерні завдання

* Розширення можливостей репозиторіїв для підтримки довготривалого збереження стану (Persistence Layer).
* Впровадження підсистеми декларативних запитів (LINQ) для аналізу та агрегації доменних даних.
* Масштабування бізнес-логіки прикладного шару із застосуванням поведінкових патернів проектування (Observer).
* Реалізація повного циклу модульного тестування (Unit Testing) для забезпечення регресійної стійкості.

### 2.2. Специфікація функціональних вимог

| Ідентифікатор | Функціональний компонент | Опис бізнес-логіки |
| --- | --- | --- |
| **FR-2.1** | Серіалізація стану подій | Асинхронне збереження та відновлення графа об'єктів системи у текстовому форматі JSON. |
| **FR-2.2** | Динамічний пошук та фільтрація | Вибірка об'єктів `Event` за предикатами (категорія, часовий діапазон, наявність вільних місць). |
| **FR-2.3** | Статистична агрегація даних | Обчислення сумарної місткості відкритих подій та визначення ТОП-популярних заходів на основі кількості реєстрацій. |
| **FR-2.4** | Поведінкові сповіщення | Автоматичне тригерення подій розсилки повідомлень примусового скасування заходів (`CancelEvent`). |
